# API Data Layer

> Generated: May 9, 2026 | Branch: development | Commit: 07478fe

## Overview

The API uses PostgreSQL 15+ as the primary data store, with Drizzle ORM handling all database interactions. The schema consists of 30+ tables organized into domains: users, authentication, conversations, memory, skills, and integrations. Semantic search is powered by embeddings (stored in vector columns and synced with Cloudflare AI). The system uses connection pooling via a configured pool size and relies on Bull + Redis for async job processing.

**Key design decisions:**
- JSONB columns store structured metadata (conversation context, memory annotations)
- Migrations are version-controlled in `drizzle/` directory
- Cascading deletes are minimal to prevent accidental data loss
- Unique constraints prevent duplicate accounts from OAuth providers

## Database schema

### Core Domain: Users & Auth

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| **users** | User accounts | `id` (uuid), `email` (unique), `name`, `createdAt` |
| **user_channels** | Platform connections | `userId`, `channelType` (telegram/whatsapp/discord), `channelId` (unique per platform) |
| **authentications** | Better-Auth sessions | `id`, `userId`, `sessionToken`, `expiresAt` |
| **oauth_accounts** | OAuth provider links | `userId`, `provider` (google/github), `providerAccountId` (unique) |
| **user_preferences** | Personal settings | `userId`, `theme`, `language`, `notifications` (JSONB) |

**Key constraints:**
- `oauth_accounts` has unique index on `(provider, providerAccountId)` to prevent duplicate OAuth accounts
- `user_channels` has unique index on `(channelType, channelId)` to map platform IDs to users

### Domain: Conversations & Messages

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| **conversations** | Conversation sessions | `id` (uuid), `userId`, `state` (enum: active/closed), `context` (JSONB), `createdAt`, `updatedAt` |
| **messages** | Message history | `id`, `conversationId`, `author` (user/bot), `content` (text), `role` (for LLM context), `createdAt` |
| **session_transcripts** | Full conversation transcripts | `sessionId`, `transcript` (JSONB array), `metadata` (JSONB) |

**Context JSONB structure:**
```json
{
  "memory_ids": ["uuid1", "uuid2"],
  "last_intent": "search",
  "search_query": "...",
  "enrichment_cache": { "tmdb": {...} },
  "user_metadata": { "timezone": "..." }
}
```

### Domain: Memory

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| **memory_items** | User-saved content | `id` (uuid), `userId`, `type` (enum: movie/tv/video/note/link), `title`, `description`, `metadata` (JSONB), `embedding` (vector), `createdAt`, `tags` (array) |
| **memory_envelopes** | Canonical memory format | `id`, `itemId`, `envelope_type`, `serialized_content` (JSONB) |
| **semantic_external_items** | Cached external entities | `id`, `externalId` (e.g., TMDB ID), `source` (tmdb/youtube), `metadata` (JSONB), `embedding` (vector), `lastUpdated` |
| **memory_insights** | Extracted insights | `id`, `itemId`, `insight_text`, `confidence` (float), `category` (theme/summary/action) |

**Embedding strategy:**
- Embeddings generated by Cloudflare AI gateway (384-dim vectors)
- Stored in PostgreSQL `vector` column (requires pgvector extension)
- Indexed with IVFFlat for semantic search performance

### Domain: Skills & Agent

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| **agent_skills** | Available skills | `id`, `name` (unique), `description`, `content` (JSONB: prompt/examples), `triggers` (array), `enabled` (boolean) |
| **agent_memory_profiles** | User memory profiles | `userId`, `profile_type`, `config` (JSONB: memory size, retention) |
| **agent_sessions** | Agent execution sessions | `id`, `userId`, `state` (enum), `startedAt`, `endedAt` |
| **agent_daily_logs** | Execution logs | `id`, `sessionId`, `date`, `events` (JSONB array) |
| **turn_audits** | Hermes kernel turn logs | `id`, `sessionId`, `turnNumber`, `toolsUsed` (array), `policies` (array), `runType` (completed/interrupted), `timestamp` |

### Domain: Integrations & Settings

| Table | Purpose | Key Columns |
|-------|---------|-------------|
| **whatsapp_settings** | WhatsApp configuration | `userId`, `phoneNumberId`, `businessAccountId`, `accessToken` |
| **integrations** | Third-party service links | `userId`, `serviceType` (spotify/youtube), `credentials` (encrypted JSONB), `scope` (oauth scopes), `expiresAt` |
| **provider_keys** | BYOK provider credentials | `userId`, `providerType` (gemini/claude), `encryptedKey` (encrypted), `model`, `isActive` |
| **providers** | Dynamic provider registry | `id`, `name`, `type`, `endpoint`, `config` (JSONB) |
| **model_registry** | AI model registry | `id`, `modelName` (e.g., gemini-pro), `provider`, `capabilities` (array), `costPerMTok` |
| **feature_flags** | Feature flags | `name` (unique), `enabled`, `channels` (array), `rollout_percentage`, `config` (JSONB) |
| **linking_tokens** | OAuth linking tokens | `token` (unique), `userId`, `provider`, `expiresAt` |

## Migrations

Migrations are version-controlled in `drizzle/` (20+ numbered SQL files). Each migration:

1. **Created via:** `pnpm run db:generate` (auto-generates from schema changes)
2. **Applied via:** `pnpm run db:push` (in dev) or `drizzle-kit migrate:prod` (in production)
3. **Named:** `NNNN_[description].sql`

**Example:**
```sql
-- 0001_volatile_rockslide.sql
CREATE TABLE users (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  email text UNIQUE NOT NULL,
  name text,
  created_at timestamp DEFAULT now()
);

CREATE TABLE oauth_accounts (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  provider text NOT NULL,
  provider_account_id text NOT NULL,
  UNIQUE(provider, provider_account_id)
);
```

## ORM patterns

### Query pattern

All queries use Drizzle query builder:

```ts
// apps/api/src/db/index.ts
import { drizzle } from 'drizzle-orm/postgres-js';
import postgres from 'postgres';

const queryClient = postgres(process.env.DATABASE_URL!);
export const db = drizzle(queryClient);
```

### Selecting

```ts
// Simple select
const user = await db.select().from(users).where(eq(users.id, userId));

// With joins
const memories = await db
  .select({ item: memoryItems, user: users })
  .from(memoryItems)
  .innerJoin(users, eq(memoryItems.userId, users.id))
  .where(eq(memoryItems.userId, userId));

// With ordering and limit
const recent = await db
  .select()
  .from(memoryItems)
  .orderBy(desc(memoryItems.createdAt))
  .limit(10);
```

### Inserting

```ts
// Single insert
const newItem = await db
  .insert(memoryItems)
  .values({ userId, type: 'movie', title, metadata })
  .returning();

// Bulk insert
await db
  .insert(messages)
  .values([
    { conversationId, author: 'user', content: msg1 },
    { conversationId, author: 'bot', content: msg2 },
  ]);
```

### Updating

```ts
// Update with conditions
await db
  .update(conversations)
  .set({ state: 'closed', updatedAt: new Date() })
  .where(eq(conversations.id, conversationId));
```

### Deleting

```ts
// Soft delete (set flag)
await db
  .update(memoryItems)
  .set({ deletedAt: new Date() })
  .where(eq(memoryItems.id, itemId));

// Hard delete (rare)
await db.delete(memoryItems).where(eq(memoryItems.id, itemId));
```

### JSONB queries

```ts
// Select with JSONB column
const user = await db
  .select()
  .from(conversations)
  .where(eq(conversations.id, convId));

// Access nested JSONB in Drizzle
const context = user[0].context; // Already parsed JSON object

// Update JSONB
await db
  .update(conversations)
  .set({ context: { ...oldContext, memoryIds: [...newIds] } })
  .where(eq(conversations.id, convId));
```

### Vector/Embedding queries

```ts
// Insert with embedding
await db
  .insert(memoryItems)
  .values({
    userId,
    title,
    embedding: embeddingVector, // 384-dim array
  });

// Semantic search using cosine distance
const results = await db
  .select()
  .from(memoryItems)
  .orderBy(
    asc(sql`1 - (embedding <=> ${queryEmbedding}::vector)`) // cosine similarity
  )
  .where(eq(memoryItems.userId, userId))
  .limit(10);
```

## Caching strategy

### Cache layers

| Layer | Tech | TTL | Purpose |
|-------|------|-----|---------|
| **Application** | Node.js memory | 5 min | Tool schemas, skill definitions |
| **Session** | Redis (Bull) | 1 hr | Conversation session state |
| **External data** | Redis + DB cache | 24 hr | TMDB metadata, YouTube results |
| **Embeddings** | Redis (Cloudflare cache) | 30 days | Query embeddings |

### Invalidation triggers

- **Memory item created:** Invalidate user's memory search cache
- **Skill enabled/disabled:** Invalidate tool schema cache
- **User preferences changed:** Invalidate preference cache
- **OAuth provider reconfigured:** Invalidate credential pool

## Connection management

### Pool configuration

```ts
// Database connection pooling
const queryClient = postgres(process.env.DATABASE_URL!, {
  max: 25, // Max concurrent connections
  idle_timeout: 30, // Close idle connections after 30s
  connect_timeout: 10,
});
```

### Health checks

Health endpoint verifies database connectivity:

```ts
// routes/health.ts
app.get('/health', async (c) => {
  try {
    await db.select().from(users).limit(1); // Quick query
    return c.json({ status: 'ok' });
  } catch (error) {
    return c.json({ status: 'error' }, 503);
  }
});
```

## Data access patterns

### Read patterns

1. **Direct lookup:** User/conversation/memory by ID
2. **Search:** Semantic (embeddings) + keyword hybrid
3. **Aggregation:** Count memories by type, analytics

### Write patterns

1. **Append-only:** Messages always inserted (never updated)
2. **Immutable snapshots:** Conversation transcripts (no updates)
3. **Idempotent updates:** Conversation context (safe to retry)

### Update patterns

```ts
// Safe update pattern (use atomic transactions if available)
const currentContext = await db
  .select({ context: conversations.context })
  .from(conversations)
  .where(eq(conversations.id, convId));

const updated = { ...currentContext[0].context, lastIntent: 'search' };

await db
  .update(conversations)
  .set({ context: updated })
  .where(eq(conversations.id, convId));
```

## Known issues & gotchas

1. **JSONB mutations:** Always spread/clone JSONB before updating; Drizzle doesn't track nested mutations
2. **Vector type:** Requires `pgvector` extension in PostgreSQL; ensure installed before running migrations
3. **Cascading deletes:** Minimal use; data archival preferred over hard deletion
4. **Connection timeouts:** Monitor pool idle timeout in high-traffic scenarios
5. **Embedding sync:** External embeddings may drift from DB values; consider scheduled sync job
